# OctoAcme Role-Specific Onboarding Checklist

## Purpose
This checklist helps new team members understand their role, responsibilities, and how they interact with other roles in an OctoAcme project.

---

## General Onboarding (All Roles)

- [ ] Read the OctoAcme Project Management Overview
- [ ] Review the Roles and Personas document to understand your role and others
- [ ] Study the Role Interaction Matrix to understand your handoff points
- [ ] Schedule 1-on-1s with key collaborators from other roles
- [ ] Access and bookmark the project repo, board, and communication channels
- [ ] Understand escalation paths and decision-making authority (see Role Interaction Matrix)
- [ ] Review the current project charter and roadmap

---

## Developer Onboarding

**Getting Started**
- [ ] Clone the repository and set up local development environment
- [ ] Review the branching strategy and PR guidelines
- [ ] Familiarize yourself with the tech stack and architecture docs
- [ ] Run tests locally and verify CI/CD pipeline access

**Understanding Responsibilities**
- [ ] Review acceptance criteria and Definition of Done
- [ ] Understand how to write tests and maintain test coverage
- [ ] Learn the code review process and feedback expectations
- [ ] Know who the System Architect is and how to reach out for design questions

**Team Collaboration**
- [ ] Attend daily standup and ask about current blockers
- [ ] Schedule a design walkthrough with the UX Designer (if applicable)
- [ ] Meet with System Architect to understand architecture decisions
- [ ] Understand how to report blockers to the Scrum Master

**First Week Goals**
- [ ] Complete a small PR to familiarize yourself with the workflow
- [ ] Identify and ask questions about unclear requirements
- [ ] Establish communication with your code review partners

---

## Product Manager Onboarding

**Getting Started**
- [ ] Review the Project Charter and success metrics
- [ ] Understand the customer problem and market context
- [ ] Familiarize yourself with the current roadmap and backlog
- [ ] Get access to analytics, customer feedback channels, and user research tools

**Understanding Responsibilities**
- [ ] Learn how backlog prioritization decisions are made
- [ ] Understand how success metrics are defined and tracked
- [ ] Review past retrospectives to understand team patterns
- [ ] Know who the Business Analyst is for requirement clarification

**Team Collaboration**
- [ ] Schedule weekly sync with Project Manager
- [ ] Meet with UX Designer to understand user research approach
- [ ] Attend sprint planning to understand team velocity and capacity
- [ ] Connect with key stakeholders and understand their needs

**First Week Goals**
- [ ] Propose one backlog prioritization change with rationale
- [ ] Schedule user research or customer interviews
- [ ] Align with Project Manager on current risks and dependencies

---

## Project Manager Onboarding

**Getting Started**
- [ ] Review the Project Charter, timeline, and key milestones
- [ ] Understand the project's risk register and mitigation plans
- [ ] Get access to project boards, scheduling tools, and reporting dashboards
- [ ] Review past project retrospectives and action items

**Understanding Responsibilities**
- [ ] Learn the escalation paths and decision-making thresholds
- [ ] Understand resource allocation and dependency management
- [ ] Know how to track and communicate project status
- [ ] Learn the risk management and blocker escalation process

**Team Collaboration**
- [ ] Schedule weekly sync with Product Manager
- [ ] Meet with Scrum Master to understand team dynamics and capacity
- [ ] Connect with System Architect on technical complexity and timelines
- [ ] Establish communication with all key stakeholders

**First Week Goals**
- [ ] Update the risk register with current assessment
- [ ] Schedule the next stakeholder status update meeting
- [ ] Identify one process improvement opportunity

---

## Scrum Master Onboarding

**Getting Started**
- [ ] Review the agile framework and ceremonies being used (Scrum, Kanban, hybrid)
- [ ] Access sprint planning tools and burndown tracking systems
- [ ] Understand the team's Definition of Done and sprint conventions
- [ ] Review past retrospectives to understand team dynamics

**Understanding Responsibilities**
- [ ] Learn the standup format and frequency
- [ ] Understand how to identify and escalate blockers
- [ ] Know the process for facilitating sprint planning, review, and retrospective
- [ ] Understand how to track team velocity and capacity

**Team Collaboration**
- [ ] Introduce yourself to each team member in 1-on-1 conversations
- [ ] Meet with Project Manager to understand project timelines
- [ ] Understand how to escalate impediments to the right person
- [ ] Connect with System Architect on technical blockers

**First Week Goals**
- [ ] Facilitate your first standup and gather feedback
- [ ] Identify one process improvement suggestion from the team
- [ ] Establish trust with team members through active listening

---

## UX Designer Onboarding

**Getting Started**
- [ ] Review the product vision and user personas
- [ ] Understand the current design system and component library
- [ ] Access design tools (Figma, Adobe XD, etc.) and brand guidelines
- [ ] Review past user research and testing results

**Understanding Responsibilities**
- [ ] Learn the design process and feedback loops
- [ ] Understand how to conduct user research and usability testing
- [ ] Know how to document design specs and deliverables
- [ ] Learn the timeline for design reviews and handoffs

**Team Collaboration**
- [ ] Meet with Product Manager to understand user needs
- [ ] Schedule a design review with Developers
- [ ] Connect with Business Analyst on user scenarios
- [ ] Understand how to iterate based on developer feedback

**First Week Goals**
- [ ] Conduct a design audit of current product
- [ ] Schedule user testing or interview sessions
- [ ] Document design system gaps or inconsistencies found

---

## Business Analyst Onboarding

**Getting Started**
- [ ] Review the Project Charter and business objectives
- [ ] Understand the current process flows and systems
- [ ] Access requirements documentation and historical artifacts
- [ ] Familiarize yourself with stakeholder groups and their needs

**Understanding Responsibilities**
- [ ] Learn how requirements are gathered and documented
- [ ] Understand the scope and acceptance criteria definition process
- [ ] Know how to conduct stakeholder interviews and analysis
- [ ] Learn the UAT process and how to support testing

**Team Collaboration**
- [ ] Meet with Product Manager to understand product vision
- [ ] Connect with Developers to understand implementation approach
- [ ] Schedule requirements review session with stakeholders
- [ ] Understand how to work with QA on test scenarios

**First Week Goals**
- [ ] Document current requirements for key features
- [ ] Identify one requirements clarity gap and propose solution
- [ ] Schedule stakeholder interviews to validate understanding

---

## System Architect Onboarding

**Getting Started**
- [ ] Review the current system architecture and design documents
- [ ] Understand the technology stack and infrastructure
- [ ] Get access to architecture documentation and decision logs
- [ ] Review past architectural decisions and trade-offs

**Understanding Responsibilities**
- [ ] Learn the process for evaluating technology choices
- [ ] Understand how to assess technical feasibility
- [ ] Know how to document architectural decisions
- [ ] Learn how to mentor teams on architectural principles

**Team Collaboration**
- [ ] Meet with Developers to understand current challenges
- [ ] Connect with Project Manager on timeline impact of architecture
- [ ] Schedule design review with System Design team (if applicable)
- [ ] Understand escalation path for major architectural decisions

**First Week Goals**
- [ ] Review and audit current architecture against best practices
- [ ] Identify one technical debt item and propose mitigation
- [ ] Schedule architecture review sessions with the team

---

## Common First-Week Activities (All Roles)

- [ ] Attend team standup or meeting
- [ ] Participate in sprint planning or backlog refinement
- [ ] Meet individually with 3-5 key collaborators from other roles
- [ ] Read and understand the project's problem statement and success metrics
- [ ] Ask clarifying questions about your role and responsibilities
- [ ] Introduce yourself to the team
- [ ] Schedule weekly 1-on-1 with your direct manager or team lead
